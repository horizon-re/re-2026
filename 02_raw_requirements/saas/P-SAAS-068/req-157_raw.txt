## Architecture / アーキテクチャ

**English:** The solution’s deployment model and end-to-end operation are outlined below:

1. **Cloud Deployment:** The system is deployed on a scalable cloud infrastructure, ensuring high availability and performance. All components run on the server side in the cloud, and the environment utilizes load balancing and auto-scaling to handle varying user traffic.
2. **Server-Side Content Execution:** When a learner launches a course, the SCORM 1.2 or xAPI content is loaded and executed entirely on the server. A WebAssembly-based content player processes the course logic and user interactions on the server. In other words, any code within the course (such as quiz logic or simulations) runs in a secure WebAssembly sandbox on the server, and the server generates the resulting HTML interface for the learner.
3. **No Client-Side JavaScript:** The user’s browser receives fully rendered HTML for each page of the training content. When the learner interacts (for example, submits an answer or navigates to the next module), those actions are sent back to the server via standard HTTPS requests. The WebAssembly engine then updates the course state on the server and the server responds with a new HTML page or snippet. This ensures the platform functions correctly even in locked-down corporate environments or on older browsers.
4. **SCORM 1.2 Tracking:** The platform includes a server-side implementation of the SCORM 1.2 API. As the content runs on the server, it calls the SCORM API internally to track learner progress – recording completion status, scores, time spent, and interaction details. This tracking data is stored securely in the platform’s database and can be synchronized with an existing Learning Management System (LMS) if required. Learner progress and results are captured reliably without any client-side scripting.
5. **xAPI Event Recording:** For xAPI-based content, the server captures learning experience events and generates corresponding xAPI statements. These statements are sent to an integrated Learning Record Store or forwarded to an external LRS as needed. Because all interactions are mediated by the server, every xAPI statement is recorded even if a user temporarily loses connectivity (the system queues and sends statements once the connection is restored).
6. **Security and Scalability:** All data transmissions are protected via SSL encryption. The platform runs in a secure cloud network environment (for instance, within a VPC with restricted access). Each component is containerized for consistency and can scale horizontally to accommodate more users. Executing training logic in WebAssembly on the server provides an extra layer of security through sandboxing. Additionally, server-side rendering means minimal sensitive data is exposed to the client. The architecture supports high availability and disaster recovery measures, including regular backups and failover capabilities to ensure continuity of service.
